# eBPF 可观测性学习路线

## 阶段 1：进程观测

目标是打通最小 eBPF 开发闭环：

- 使用 Go 加载 eBPF 程序。
- 使用 C 编写最小内核态 eBPF 逻辑。
- 监听进程 exec 或 exit 事件。
- 从内核态向用户态传递 PID、UID、命令名、参数等事件数据。
- 在 Go 侧读取事件并格式化输出。

建议先理解这些概念：

- tracepoint、kprobe、fentry 的差异。
- ring buffer 和 perf event 的使用场景。
- Go 结构体和 C 结构体的内存布局对齐。
- eBPF 程序需要的 Linux 权限和内核版本约束。

## 阶段 2：网络观测

目标是在进程观测的基础上扩展到 TCP 连接链路：

- 观测 TCP connect、accept 或 close 事件。
- 采集本地地址、远端地址、端口、PID、进程名。
- 区分连接成功、失败和关闭。
- 将网络事件和进程信息关联起来。

建议先做最小闭环：

- TCP connect 事件观测。
- 输出 PID、进程名、目标 IP、目标端口。
- 再逐步加入耗时、失败原因和连接生命周期。

## 阶段 3：扩展方向

完成进程和网络观测后，再考虑：

- 系统调用观测：open、read、write、connect。
- 文件 IO 观测：文件路径、读写大小、耗时。
- 性能观测：CPU profile、off-CPU、调度延迟。
- 输出增强：表格、JSON、过滤条件、采样。

## 工程原则

- Go 优先，C 辅助。
- 每个主题先在 `examples/` 中形成最小可运行示例。
- 示例稳定后再抽取公共代码到 `internal/`。
- 不提前做复杂抽象，优先保证学习路径清晰。
